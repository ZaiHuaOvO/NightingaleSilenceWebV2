# NSArmoire 衣柜清理大师

本文件为#/ffxiv/armoire → NSArmoire项目的需求文档。


## 需求

1.以网页+本地助手形式，用户点击本地助手可以读取到游戏内的物品栏（包含：背包、陆行鸟鞍囊、雇员背包、兵装库、投影台、收藏柜）
2.用户点击本地助手的查看即可跳转到网页检查自己的仓库
3.本工具需要提前载入最终幻想14游戏客户端数据，找到所有可放入投影台、收藏柜的外观物品，
4.可以向用户展示收藏柜收集进度，显示用户未收藏物品，并引导用户补全
5.可以向用户展示自己的物品收集系统中外观的分布情况，有多少分布在背包、有多少分布在雇员、等等……
6.本工具参考 https://github.com/Asvel/ffxiv-dresser-analyze 的功能，希望可以实现其同原理功能，例如可以向用户展示多少现有外观可以被套装幻影化或进入收藏柜，例如检测装备是否被染色，若染色了则不推荐放入收藏柜或进行套装幻影化，以及同模型装备检测

## 已确认规则

- 投影台里存在 `ItemUICategory=112` 的套装幻影化物品时，不代表用户已经完全收集了该套装内的全部散件。
- `112` 物品应理解为一个套装收纳容器/篓子：它可以处于残缺状态，内部散件需要分别判断是否已收集。
- 计算套装收集度时，不允许仅凭用户拥有 `112` 套装物品就把 `MirageStoreSetItem.csv` 中关联的全部散件标记为已收集。
- NSArmoire 的收集度分成两个独立口径，不合并成一个总收集度：
  - 收藏柜收集度：收藏柜现有个数 / 可放入收藏柜的个数。
  - 投影台套装投影化进度：投影台已套装投影化个数 / 可套装投影化的个数。
- 投影台统计需要展示已经套装投影化但缺件的套装，并明确缺少哪些散件。

## 后续想法：卫月插件路线

- 未来可以尝试为 NSArmoire 单独编写卫月 / Dalamud 插件，用插件读取游戏内物品状态，再输出或提供给网页分析。
- 该插件暂时不放在 V2 仓库中，应另开独立项目规划和实现。
- 真正启动该方向前，需要先阅读官方 Dalamud 开发者指南，确认插件模板、API 能力、数据读取边界、分发方式和安全要求。
- 当前开发顺序不因该想法改变：V2 先继续按既定顺序推进 NSPlate / NSGlamour 主线；NSArmoire 插件方向作为后续预研记录。

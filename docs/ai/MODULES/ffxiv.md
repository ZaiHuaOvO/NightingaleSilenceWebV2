# FFXIV 模块计划

## 当前状态

- 模块状态：占位骨架已接入，真实工具业务待迁移。
- 计划路由：`#/ffxiv`。
- 计划页面入口：`src/pages/ffxiv/FfxivIndexPage.vue`。
- 模块类型：分类导航页 / 工具入口页。
- 当前不涉及后端 API。

## 定位

`#/ffxiv` 是《最终幻想14》相关工具的分类导航页。它承接首页的 `FFXIV` 主入口，再把用户导向具体工具页面。

`FFXIV` 是当前第一阶段分类，不代表整站长期只服务《最终幻想14》。未来其他游戏工具、博客或创作信息应作为新的分类或内容模块扩展。

路由层级：

```text
#/                 Home 纯视觉首页
#/ffxiv            FFXIV 分类导航页
#/ffxiv/glamour    NSGlamour 幻化工具
#/ffxiv/plate      NSPlate 铭牌工具
```

## 与首页的分工

首页 `#/`：

- 负责品牌第一印象。
- 以人物静止图、浮动图标、少量按钮为主。
- 只保留一个清晰的 `FFXIV` 主入口。
- SNS 外链可以作为边角轻量元素存在。

FFXIV 分类页 `#/ffxiv`：

- 负责工具导航。
- 展示 `glamour`、`plate` 和未来更多 FFXIV 工具。
- 可以比首页更信息化，但仍保持视觉统一。
- 不承担具体工具的复杂编辑功能。

## 页面结构建议

```text
FfxivIndexPage
├── 分类页标题：FFXIV
├── 简短副标题：数据工具入口
├── 工具入口网格
│   ├── NSGlamour 工具卡
│   └── NSPlate 工具卡
└── 返回首页 / 外链辅助区
```

分类页不需要复制首页的大型人物舞台。可以保留少量视觉元素，如低透明背景纹理、图标、光晕或细线框架，让它和首页同属一个视觉系统。

## CSS 和组件策略

- 样式层级：模块/分类级。
- 工具卡、按钮、面板、标签、状态文本优先使用公共组件和公共 CSS。
- 页面可以有自己的分类页布局，但不要复制首页人物舞台样式。
- 不为 FFXIV 分类页单独建立主题，继续使用 `atelier` 变量体系。
- 若工具卡未来也用于其他分类，应抽成公共 `ToolCard` 或类似组件。
- 首页粉蓝像素风可以作为轻微色彩呼应，但窗口贴纸、人物舞台、强像素背景、浮动图标轨道不进入 FFXIV 分类页默认样式。
- 如果 FFXIV 分类页需要特殊工具卡外观，优先放在 `src/pages/ffxiv/` 或模块私有组件，不修改公共组件默认外观。

## 待确认事项

- 工具卡展示信息：只显示名称，还是包含一句简介。
- 分类页是否显示 SNS 外链，还是只在首页显示。
- 是否需要在工具卡上显示“计划中 / 可用”状态。
- 未来 FFXIV 工具是否统一放在该页，还是某些工具会独立成其他分类。

## 验证清单

实现后至少验证：

- `npx vue-tsc --noEmit`
- `npm run build`
- 浏览器访问 `http://localhost:5173/#/ffxiv`
- 工具入口在桌面端和移动端布局稳定。
- 工具入口可键盘聚焦和回车进入。
- 未实现工具页时，占位跳转不会造成空白或路由错误。

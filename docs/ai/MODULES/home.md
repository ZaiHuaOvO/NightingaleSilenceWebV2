# Home 模块计划

## 当前状态

- 模块状态：占位骨架已接入，最终视觉和真实内容待完善。
- 计划路由：`#/`。
- 计划页面入口：`src/pages/home/HomePage.vue`。
- 来源项目：`NSHome`，但 V2 首页不是简单搬旧静态页，而是重做为纯视觉入口页。
- 当前不涉及后端 API。

## 设计目标

首页暂定为“纯视觉入口页”：第一屏以人物静止图为主体，配合浮动图标和少量入口按钮，形成夜莺不语网站的视觉门面。

核心气质：

- 视觉强、仪式感强，不做传统营销落地页。
- 首页代表夜莺不语个人/工具网站整体，不是 FFXIV 专属首页。
- 中央人物是主角，周围图标是功能星群。
- 第一版使用顶部 bar 作为主要导航，保留少量入口。
- 顶部 bar 放 `狒狒14工房`、`Silence` 和 `关于 | ABOUT`。
- `狒狒14工房` 进入 `#/ffxiv` 分类导航页。
- 后续可扩展其他分类；当前已确认 `ffxiv` 和 `silence` 两个公开分类入口，其中 `silence` 已接入双入口门厅。

视觉风格：

- 粉色 + 蓝色的可爱主播女孩像素风格。
- 参考用户皮套形象：粉发、浅蓝/薄荷色、天使光环/翅膀、耳机、游戏手柄、窗口、文件夹、爱心、鼠标指针等元素。
- 色彩要偏柔和，避免高饱和荧光粉/荧光蓝造成刺眼。
- 可以使用像素风边框、点阵纹理、窗口 UI、游戏 UI 和轻量 glitch/error 元素，但整体保持清爽。

当前参考效果：

- Aceternity `Parallax Hero Images` 的鼠标视差浮动感。
- 不直接使用它的 React / Tailwind 实现，只复刻“分层浮动”的运动感觉。
- 顶部 bar 位置和轻量导航气质参考 `https://ak.hypergryph.com/#index` 的右上角导航。

## 视觉结构

页面结构建议：

```text
HomePage
├── 顶部 bar：右上导航，包含 狒狒14工房 / 关于 | ABOUT
├── 背景层：低透明纹理、光晕、星点或抽象网格
├── 人物层：中央透明 PNG/WebP 静止图
├── 图标层：围绕人物的功能入口图标
├── 文案层：NIGHTINGALE SILENCE / 夜莺不语
└── 辅助入口层：SNS/Links 等后续可选
```

第一版只需要人物静止图，不使用 Live2D、VRM、Three.js 或复杂动画库。

## 顶部 bar

第一版首页使用一个头顶/顶部 bar，视觉上靠近画面右上角，参考 `https://ak.hypergryph.com/#index` 的右上角导航密度。

导航项：

| 文案            | 目标                             | 状态               |
| --------------- | -------------------------------- | ------------------ |
| `狒狒14工房`    | `#/ffxiv`                        | 确认               |
| `Silence`        | `#/silence`                      | 确认               |
| `关于 \| ABOUT` | 待定：可以先做锚点、弹窗或占位页 | 确认显示，目标待定 |

设计要求：

- bar 不能压过人物脸部和主视觉。
- 体量要轻，像头顶导航而不是大面积菜单。
- 文字需要清楚可读，不能只做图标。
- 桌面端可放右上角；移动端需要收敛成顶部横条或紧凑按钮组。
- hover 可以有像素风高亮、浅蓝/粉色描边、轻微位移或发光，但不要闪烁刺眼。

## 两套浮动逻辑

### 人物层：稳、慢、有重量

人物是视觉中心，运动必须克制：

- 入场动画：从略低处进入正中，`opacity` 从 0 到 1，`blur` 从轻微模糊到清晰，`scale` 从约 0.96 到 1。
- 待机动画：极慢上下漂浮，幅度约 4-8px，周期约 5-8 秒。
- 鼠标视差：人物只做小幅位移和小角度旋转，避免晕动。
- 插值方式：不直接跟随鼠标，使用 `requestAnimationFrame` 做缓慢追随。
- 目标感觉：人物像舞台中心的稳定主角，而不是被鼠标甩动的卡片。

建议上限：

| 属性         | 上限           |
| ------------ | -------------- |
| `translateX` | 约 8-12px      |
| `translateY` | 约 8-12px      |
| `rotate`     | 约 0.5-1deg    |
| `scale`      | 常态保持接近 1 |

### 图标层：轻、分层、漂浮

图标承担装饰和入口暗示，可以更接近 Aceternity 的浮动感：

- 每个图标有独立深度系数。
- 鼠标移动时，图标按不同速度和方向移动。
- 部分图标可以反向移动，增强空间感。
- 待机时各自有不同周期的轻微漂浮。
- hover 时可轻微放大、提亮、发光。
- 点击后进入对应路由；首页图标优先指向 `#/ffxiv`，具体工具入口放到分类页。

建议图标分层：

| 层       | 运动幅度 | 用途             |
| -------- | -------- | ---------------- |
| 远景图标 | 大       | 主要营造空间     |
| 中景图标 | 中       | 暗示分类和工具   |
| 前景入口 | 小到中   | 可点击按钮或图标 |

## 资产要求

人物资产：

- 透明背景人物图，优先 `webp`，可保留 `png` 作为源文件。
- 高度建议 1800px 以上，方便高清屏缩放。
- 人物边缘需要干净，不要带白边或黑边。
- 最好额外提供一张人物背后的光晕/阴影层，可独立动效。

背景资产：

- 可以先用 CSS 渐变、噪声纹理、星点和光晕实现。
- 不使用大体积视频作为第一版背景。
- 不使用会喧宾夺主的高对比背景图。

图标资产：

- 优先 SVG 或轻量 PNG/WebP。
- `ffxiv`、`glamour`、`plate` 后续可做成统一风格的功能图标。
- 图标必须有文本标签或可访问名称，不能只靠图形表达。
- 风格方向已确认：可爱像素 UI，优先使用窗口、文件夹、爱心、游戏手柄、鼠标指针、星星、光环/翅膀等与用户皮套一致的符号。

## 组件拆分计划

建议页面结构：

```text
src/pages/home/
├── HomePage.vue
└── components/
    ├── HomeTopBar.vue
    ├── HomeCharacterStage.vue
    ├── HomeFloatingIconField.vue
    ├── HomeCategoryEntry.vue
    └── HomeVisualTitle.vue
```

可复用逻辑：

```text
src/composables/
└── usePointerParallax.ts
```

如果后续其他页面也需要鼠标视差，`usePointerParallax.ts` 应作为公共 composable；如果只有首页使用，可以先放在 `src/pages/home/` 内部，等第二处复用再上移。

## CSS 和组件复用策略

遵守 `ARCHITECTURE_PLAN.md` 的“样式变更分级规则”：

- 全站颜色、字体、按钮、面板、状态反馈使用公共 CSS 和公共组件。
- 首页特殊的舞台布局、人物定位、浮动图标轨道，可以写在页面 scoped style。
- 首页顶部 bar、按钮、链接如果与其他页面共用，应抽成公共按钮组件或公共 class。
- 不为首页单独建立新主题；继续使用全站像素风变量体系。
- 首页可以在全站像素风变量基础上派生更强的粉蓝人物舞台变量，但不要另建主题系统。
- 首页像素边框、窗口贴纸、人物舞台、浮动图标、强粉蓝背景都属于页面专属样式，不进入全站公共组件默认外观。
- 如果首页按钮使用公共按钮组件，只通过局部 modifier 或 wrapper 做首页皮肤，不修改公共按钮默认样式。
- 当前首页视觉方向已开始贴近 Style Lab 的 `pixel-soft`：硬边框、方角窗口、粉蓝像素标题、窗口 chrome 和少量像素浮动点缀。实现时使用首页 scoped style 复刻视觉语言，不直接把 `src/styles/experiments/pixel-soft.css` 导入正式首页。

## 动效实现策略

第一版不引入新依赖：

- 入场动画用 CSS keyframes 或 transition。
- 待机漂浮用 CSS animation。
- 鼠标视差用 Vue 事件 + `requestAnimationFrame` 插值。
- 所有动画优先使用 `transform` 和 `opacity`，避免触发布局重排。

需要支持：

- `prefers-reduced-motion: reduce`：关闭或大幅降低漂浮和鼠标视差。
- 移动端：没有鼠标时使用低幅度自动漂浮，不依赖 hover。
- 键盘导航：入口按钮必须可聚焦、可回车进入。

## 性能边界

- 第一版不使用 Live2D、Three.js、视频背景和大型粒子库。
- 人物图和背景图需要控制体积，优先 WebP。
- 不在 `mousemove` 里直接频繁写 DOM 样式，统一走 RAF。
- 不让图标数量过多，避免首页变成性能测试页。
- 动效以“高级浮动感”为目标，不追求满屏特效。

## 可访问性和内容边界

- 首页即使偏视觉，也必须保留清晰可读的站名和入口。
- 入口按钮需要真实文本，不只放图标。
- 顶部 bar 的 `狒狒14工房` 和 `关于 | ABOUT` 必须可键盘聚焦。
- 人物图需要合适的 `alt` 或作为装饰图处理。
- 文字不能压在人物复杂区域上导致不可读。
- 视觉页不写大量功能说明，具体工具说明放到对应工具页。

- `prefers-reduced-motion` 下动画是否收敛。

- 控制台无错误。

# 工房工作台样式契约

## 定位

本文件定义 V2 中复杂工具页的工作台样式和交互边界，优先服务 `#/ffxiv/plate`、`#/ffxiv/glamour` 等工房类页面。

工房页的目标不是复刻首页的视觉舞台，而是在全站像素风基础上建立高可用、低干扰、可长时间操作的工具界面。不同工房可以拥有不同布局，但应共享同一套基础控件、状态语言、密度原则和视觉边界。

## 适用范围

适用：

- `NSPlate` 铭牌工房。
- `NSGlamour` 幻化工房。
- 未来同类“左侧预览/结果、右侧配置/数据”的工具页。
- 工具页中的 canvas、预览、表单、列表、导入、导出、状态提示、弹窗和工具栏。

不适用：

- `#/` 首页的人物舞台、浮动图标、贴纸和强视觉动效。
- `#/ffxiv` 分类导航页的工具入口卡片。
- `Style Lab` 的实验皮肤。
- 单次临时调试页。

## 核心原则

1. **可用性优先于装饰**：工房页是高频操作界面，首要目标是看得清、点得准、状态明确、长时间用不累。
2. **全站统一控件，页面自定布局**：按钮、输入框、tab、toolbar、status、dialog 等基础控件尽量统一；具体左右栏、tab 数量和业务面板结构由各工具决定。
3. **像素风克制使用**：像素边框、窗口标题栏、硬阴影和 active 状态可以体现站点气质；大量数据、缩略图、装备名和参数表单要保持清爽。
4. **不把首页舞台带进工作台**：首页可以可爱、炫酷和装饰丰富；工房页只继承色彩、字体、边框和控件语言。
5. **先规范，后抽象**：不要过早建立通用工作台布局组件。先让 `NSPlate`、`NSGlamour` 按同一套规范分别完成，确认重复结构后再抽到 `src/pages/ffxiv/components/`。

## 样式层级

### 全站公共层

由 `src/styles/` 和 `src/components/` 维护，所有页面可用。

包括：

- 全站字体、字号层级、语义色、day/night 配色变量。
- `AppButton`、`AppField`、`AppToolbar`、`AppTabs`、`AppStatus`、`AppPixelWindow` 等基础组件。
- 基础 focus、hover、active、disabled、loading、error 状态。

规则：

- 工房页优先使用公共组件和公共变量。
- 需要改公共控件时，必须确认影响首页、分类页和至少一个工房页。
- 不为单个工房页修改公共组件默认外观；需要局部差异时用 prop、modifier 或页面私有样式。

### 工房工作台层

用于 `FFXIV` 工具页的共享视觉规范，但不一定对应一个共享 Vue 组件。

建议沉淀的变量方向：

```css
--workbench-bg
--workbench-panel-bg
--workbench-panel-border
--workbench-control-bg
--workbench-control-height
--workbench-active
--workbench-muted
--workbench-danger
--workbench-shadow
--workbench-gap
```

规则：

- 变量可放在全站 `theme.css` 或未来 FFXIV 模块样式中，但使用前要确认不会污染首页舞台。
- 工房页背景应低饱和、低对比，避免压过画布、截图、装备名和缩略图。
- 控件高度、边框、间距和 active 色应相近，使不同工房看起来属于同一站点。

### 工具页面私有层

由具体工具维护，例如：

- `src/pages/plate/components/`
- `src/pages/glamour/components/`
- `src/lib/plate/`
- `src/lib/glamour/`

包括：

- 画布尺寸、预览比例、拖拽区域、裁切区域。
- 右侧面板结构、业务 tab、数据列表、装备栏、模板栏。
- 特定导入、导出、图片槽、信息图层、染剂选择等业务控件。

规则：

- 页面私有样式只处理业务布局，不复制公共按钮、输入框和弹窗样式。
- 如果同一结构在 `NSPlate` 和 `NSGlamour` 中都稳定出现，再评估上提到 `src/pages/ffxiv/components/`。

## 工作台结构建议

桌面端常见结构：

```text
AppTopNav
└── Workbench
    ├── 主区域：canvas / preview / result
    ├── resize handle（如果需要）
    └── 配置区域：tabs / forms / lists / actions
```

可选结构：

- 顶部或底部局部工具栏：缩放、清空、导入、导出、状态。
- 右上浮层：设置、下载、配置导入导出。
- 弹窗：裁切、确认、批量导入、错误详情。
- 抽屉或折叠区：长列表、过滤器、历史记录。

不要：

- 不要把工具页包进大型说明卡片。
- 不要在工作台默认显示迁移统计、API 调试、内部字段盘点。
- 不要做多层卡片套卡片，尤其是右侧高密度配置区。
- 不要把主要操作藏在很像装饰的图形里。

## 控件和密度

按钮：

- 主要命令使用 `AppButton` 或同类公共按钮。
- 工具栏图标按钮必须有 `aria-label` 或可见文字。
- 危险操作使用明确语义色和确认流程。

输入框：

- 使用 `AppField` 或公共输入样式。
- 标签、错误、帮助信息要靠近输入项。
- 长文本、装备名、模板名允许换行或省略，但不能压缩字体导致难读。

Tabs：

- 用于主工作流分区，例如 `肖像 / 铭牌 / 信息` 或 `装备 / 模板 / 导入`。
- tab 数量过多时改用分组、下拉或二级区域，不要硬塞一排。

工具栏：

- 同类操作放一起，例如导入、清空、识别；导出、复制、下载。
- 工具栏不承担说明文案，说明文字应尽量进入 tooltip、status 或文档。

状态提示：

- 成功、警告、错误、加载和空状态使用统一 `AppStatus` 或公共状态样式。
- 空数据允许正常导出空图时，不要显示阻断性“等待数据”。
- 错误提示要说明用户能做什么，不暴露内部路径、堆栈或密钥。

滚动容器：

- 内部列表、弹窗正文、素材面板、右侧配置面板等需要独立滚动时，优先给滚动容器加 `.ns-scroll-area`。
- 高密度列表使用 `.ns-scroll-area--compact`。
- 不要全局重写页面滚动条；页面主滚动、浏览器默认滚动和小输入框滚动不强制像素化。

## 视觉边界

可以使用：

- 轻量像素边框。
- 方角窗口感。
- 低饱和粉蓝 day 配色。
- 低饱和紫青 night 配色。
- 小面积强调色表示 active、focus、危险和成功。

谨慎使用：

- 大面积渐变背景。
- 重阴影。
- 贴纸、网格、强纹理。
- 大标题和 hero 级文字。
- 持续动画。

不要使用：

- 首页人物舞台式装饰。
- 大面积营销 hero。
- 让缩略图、装备名、参数表单被装饰抢走注意力的背景。
- 为了“像素风”牺牲点击目标、可读性和滚动稳定性。

## 响应式规则

桌面端：

- 优先保留完整工作台能力。
- 允许左右双栏、可拖拽侧栏、密集缩略图和多列信息。
- 关键操作第一屏可见，长列表在面板内部滚动。

移动端：

- 优先上下布局：预览/画布在上，配置/列表在下。
- 隐藏或禁用桌面专属 resize handle。
- tab 可以变高、按钮可以换行，但不能出现横向溢出。
- 上传、裁切、导出等流程必须能完成；复杂批量编辑可以降级，但要说明。

## 与旧项目迁移的关系

迁移旧项目时，先分清：

- **必须还原**：用户操作路径、数据结构、画布尺寸、导出尺寸、图层顺序、素材选择、导入导出结果。
- **可以重做**：按钮外观、面板边框、颜色变量、tab 样式、弹窗视觉。
- **不能带入正式 UI**：调试字段、迁移统计、旧历史字段盘点、接口内部状态、AI 生成的正式文案。

`NSPlate` 应以旧 `NSPortable` 的双栏工作台为布局契约；`NSGlamour` 可按模板/装备/导入的业务形态自定布局，但基础控件和视觉密度应遵守本文件。

## 何时抽共享组件

暂不一开始抽通用 `WorkbenchShell`。

满足以下条件后再抽：

1. `NSPlate` 已完成真实工作台后，再等 `NSGlamour` 或第二个复杂工具完成至少一个真实业务切片。
2. 两边都出现相同结构，例如主预览区 + 右配置面板 + resize handle。
3. 共享组件不会迫使某个工具绕路实现自身业务。
4. 抽出后能减少重复样式和交互代码，而不是只为了看起来“架构整齐”。

优先抽取顺序：

1. 工作台变量和公共 class。
2. 小控件，例如工具栏按钮组、状态条、面板标题。
3. 局部容器，例如配置面板、可拖拽 handle。
4. 最后才考虑完整 `FfxivWorkbenchShell`。

## 验证清单

每次修改工房工作台样式或结构后，至少检查：

- 桌面端第一屏：主操作、右侧面板和状态是否清楚。
- 移动端：没有横向溢出，tab 和按钮可点击。
- day/night：控件、缩略图、canvas、图片不被滤镜或主题色污染。
- 长文本：装备名、模板名、素材名、错误提示不被压缩或遮挡。
- 长列表：滚动稳定，不因缩略图 lazy load 或搜索结果刷新而跳动。
- 弹窗：焦点、关闭、确认、取消和错误状态可用。
- 旧项目对照：涉及迁移还原时，至少用一个真实旧样本对比。

# 文档自愈契约

## 目标

文档必须帮助下一位 Agent 在不依赖聊天历史的情况下定位事实、完成任务并验证结果。文档不是完成感记录，也不能用过时的“已完成”描述代替当前代码检查。

## 头部摘要

`AGENT_WORKFLOW.md`、`docs/ai/*.md` 和 `docs/ai/MODULES/*.md` 的前部应使用以下字段；用户手写愿景、第三方原文和纯数据说明可以豁免：

```yaml
---
summary: "文档解决什么问题。"
status: "active | draft | legacy | archived"
scope: "覆盖哪些模块、文件或流程。"
source_of_truth: "事实应与哪些代码、配置或外部契约对照。"
read_when: "什么任务必须读。"
update_when: "哪些代码变化必须同步本文档。"
verify: "如何确认文档仍然准确。"
---
```

摘要应具体到可检索的模块名、路由或流程。不要只写“项目文档”“相关说明”。

## 事实优先级

1. 用户当前明确确认的需求和公开边界。
2. 当前代码、配置、测试和生成器的真实行为。
3. 模块专项文档和 API 契约。
4. 总览、计划和历史记录。

发现冲突时先核对高优先级来源，再修正文档；不要为了让文档“看起来一致”而改坏业务代码。

## 更新触发矩阵

| 变化 | 必须同步 |
| --- | --- |
| 路由、页面入口、模块状态 | `PROJECT_CONTEXT.md`、`MODULE_MAP.md`、对应模块文档 |
| API、端口、代理和错误契约 | `API_CONVENTIONS.md`、`docs/api/*`、对应模块文档 |
| 公共样式和组件边界 | `ARCHITECTURE_PLAN.md`、`STYLE_AUDIT.md` 或工作台契约 |
| 数据源、生成器、公开产物 | 对应模块文档、维护文档、部署检查 |
| 导入导出、草稿和兼容格式 | 对应模块文档和回归文档 |
| 验证命令或发布流程 | `AGENT_WORKFLOW.md`、`DEPLOYMENT_CHECKLIST.md` |
| 项目硬规则 | `AGENTS.md`，并检查工作流是否需要引用更新 |

## 模块文档结构

模块文档至少包含：

1. 当前状态和未完成边界。
2. 页面、路由和主要文件入口。
3. 数据源、API 和公开边界。
4. 核心业务约定与兼容风险。
5. 专项验证命令和回归样本。
6. `update_when` 中列出的自愈触发条件。

当前状态放在文档前部。历史过程只有在解释兼容决策时才保留；长时间线应整理为专题或归档，不要持续把每次 session 追加到状态段。

## 公开安全

- 不写入 token、Cookie、登录态、二维码会话、私钥和完整私有配置。
- 公共文档不写个人用户目录、本机下载路径或仅当前机器有效的绝对路径。
- 未授权图片和本地 fixture 只描述边界，不复制内容或加入构建产物。
- 本机路径、工具探测结果和账号映射分别写入 ignored `LOCAL_ENVIRONMENT.md`、`LOCAL_GIT_IDENTITIES.md`。

## 自愈流程

1. 开始任务时读取文档头，确认范围和事实来源。
2. 实现中发现漂移，记录需要同步的文档，不绕过。
3. 代码行为稳定后更新当前状态、边界和验证方式。
4. 用 `rg` 检查旧路由、旧状态、失效文件名和重复规则。
5. 收尾时在总结中说明同步了哪些文档；无法确认的内容标为待核对，不猜测。

常用检查：

```powershell
rg -n "^summary:|^status:|^read_when:|^update_when:|^verify:" AGENT_WORKFLOW.md docs/ai
rg -n "旧路由|旧文件名|待迁移|尚未实现" docs/ai
git diff --check
```

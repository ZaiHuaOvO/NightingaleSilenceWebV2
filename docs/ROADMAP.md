# Roadmap

本文件用于给项目评估和后续开发排序。模块的真实行为以当前代码、`docs/ai/MODULE_MAP.md` 和对应 `docs/ai/MODULES/` 文档为准。

## 当前阶段

阶段：V2 现有工具收口、生产边界稳定和回归补齐。

已经形成可用工作流：

- Vue 3、Vite、TypeScript、Pinia 注册、Vue Router hash 路由和公共样式/组件基础。
- 首页和 FFXIV 分类导航。
- NSPlate 核心工作台，默认使用静态 manifest + COS/CDN，支持主要 Canvas、信息层和导入导出能力。
- NSGlamour EquipInfo/Template 双工作台，共享 `GlamourDraft`，支持导入、装备/染剂编辑、六套模板、图片裁剪/暂存和 PNG 导出。
- NSArmoire 公网下载教程页，以及由 `NSArmoireButler` WPF/WebView2 GUI 内嵌的本地完整工作台。
- 时尚品鉴当前周方案、金牌物品和来源页面。
- 独立物品卡片工作台。
- Silence 入口、分组和部分角色详情代码；该模块资料尚未填写完成，暂不纳入当前公开上线范围。

## 当前优先级

1. 稳定上线边界。
   - 维护公开构建检查，防止 Armoire catalog、本地路径、私有样本或未授权资产进入产物。
   - 维护 NSGlamour 生产反向代理、外部导入和错误分支。
   - 对公开 FFXIV 页面执行生产预览和上线后烟测。

2. 补齐关键回归。
   - NSPlate 维护静态 manifest、COS 资源、Canvas、信息层和导出回归矩阵。
   - NSGlamour 维护真实导入样本、六套模板预览/原尺寸导出和移动端主流程。
   - NSArmoire 维护 Helper 同源启动、snapshot、catalog 和分析契约。
   - 时尚品鉴维护当前周公开切片、名称索引和来源证据。
   - 物品卡片维护导入、编辑和三种导出形态。

3. 控制性能和流量。
   - 继续按路由和功能区懒加载大型工作台、弹层、模板资产和 catalog。
   - 保持 NSPlate 缩略图走 WebP，Canvas 和导出按需读取原图。
   - 保持 NSGlamour 预览降采样、导出原尺寸、模板背景按需加载和搜索缓存。

4. 维护文档和数据。
   - 路由、公开边界、API、构建方式或模块状态变化时同步对应文档。
   - 时尚品鉴按周更新公开切片并保留来源核验。
   - FFXIV CSV 派生数据通过生成器和 checker 更新，不把全量源数据打入前端。

## 模块后续

### NSPlate

- 维护视觉和导出回归样本。
- 继续确认字体授权、OpenType 特性和生产缓存。
- 保持旧 `NSPortable` 服务只作为 legacy/dev fallback 和契约参考。

### NSGlamour

- 继续旧项目等价回归，不改动用户确认的装备顺序、染剂语义和模板坐标。
- 稳定旧 Flask 后端的长期部署边界；后端替换必须先通过现有 API 契约和真实样本。
- 以真实浏览器请求和 Canvas 基线评估后续 renderer 拆分，不为拆分而拆分。

### NSArmoire

- 公网只维护教程和 GitHub Release 下载入口。
- 完整工作台只进入 `armoire-local` 构建，由 Helper GUI 内嵌并同源提供 API 与 catalog。
- 继续验证游戏版本变化、容器读取、角色档案、商城目录和保守分析规则。

### 物品卡片与时尚品鉴

- 物品卡片继续维护独立数据和 UI 边界，不与 Glamour 页面互相引用私有状态。
- 时尚品鉴继续按周维护当前公开方案、名称索引和来源证据。

### Silence

- 当前代码保留作为本地开发和资料录入基础。
- 在角色资料、正式文案和授权素材完成前，不纳入公开上线范围。
- 正式恢复开发时先更新模块文档和部署清单，再进行页面与资产回归。

## 长期方向

- 逐步替换仍依赖旧服务的后端，但保持前端 API 路径和行为契约稳定。
- 完善生产 Nginx、CDN、缓存、监控和回滚流程。
- 在 FFXIV 分类稳定后扩展博客、创作信息和其他工具分类。
- Silence 资料完成并确认公开边界后，再单独制定上线计划。

## 暂缓事项

- 为统一技术栈而合并互不相关的后端。
- 在没有回归基线时改写 Canvas 视觉参数。
- 在没有明确收益和实测数据时引入新依赖或大型公共抽象。
- 在资料和资产未完成前公开 Silence。
